### v0.2 改訂の経緯

v0.1 の実装（docs/02, 05〜10）は仕様通りだったが、**画面が平面的で単調**という評価になった。原因は次の 4 点と特定した。

1. 藤紫の単色運用で、強調の手段が「藤紫にする」しかない
2. 日本語見出しが 1 行だけで、情報の階層が薄い
3. 全ページが生成り一色で、面の切り替えがない
4. タイプスケールが小さく、視線のアンカーがない

v0.2 はこの 4 点への処方として、**金の追加 / 英字併走 / 面構成 / スケール拡大**を導入する。

### 根拠となったモックアップについて

v0.2 は、ホーム / 工芸詳細 / イベントカレンダーの 3 画面の HTML モックアップを基に起こした。**モック自体は本書に蒸留したうえで削除済み**で、以後の正本は本書のみ。復元できない実データ（麻の葉パターンの SVG・ロゴのシンボルマーク）は §5.9 / §5.13 に転記してある。

モックから引き継いだ前提のうち、実装側で必ず補う必要があるもの:

- モックは**デスクトップ専用**（`min-width: 1200px`）で **SP レイアウトは存在しなかった** → §8 の規約に従って実装側で設計する
- モックは 10 画面のうち **3 画面しか無かった** → 残り 7 画面は §6「一覧ページ（モック無し・外挿ルール）」に従う
- モック内の画像は外部ホストの仮素材（出典・ライセンス不明）→ **実装に持ち込まない**。写真が入るまでは §9 のプレースホルダで構成する

## 1. デザインコンセプト

**「体験への案内係」を、藤色の糸でつなぐ。**

- Airbnb から借りるもの: 写真主導のカード、角丸の温かい UI、体験を探す→詳細→申込方法という迷わない導線、余白による信頼感
- このサイト固有のもの: 日本の伝統色（藤色 × 生成り × 藍）、明朝体の見出し、そして「糸」のモチーフ
- 対象読者は県外観光客・訪日客・地元の方・高齢の担い手。**誰が見ても威圧感がなく、温かく、迷わない**ことを最優先する

### シグネチャー要素 1: 「糸と結び目」
サイトの使命「点と点を線で結ぶ」を視覚言語にする。

- セクション区切りは水平線ではなく、**1px の細い糸（藤色）に結び目（5px の丸）を 2 つ打った線**（v0.2 で 1 つ → 2 つ。左右 25% の位置に打ち、中央を空けることで「渡した糸」に見せる）
- 工程ページのステップは、縦に垂れる 1 本の糸が各工程（結び目）を貫くタイムライン表現。結び目は**藤紫の芯 + 金の外輪**（v0.2）
- カレンダーのイベントドットも同じ「結び目」— 日々の点が糸で結ばれてゆくメタファー

### シグネチャー要素 2: 「金の栞」（v0.2 追加）

藤紫の単色運用では強調手段が乏しく、画面が平面的になっていた。**金（山吹）** を第 2 アクセントとして追加する。

- 用途は本の「しおり」に相当する**構造の目印**に限る — 章番号の罫線、英字ラベル、結び目の外輪、リンクの下線、マーカー引き
- **面として広く塗らない**。線・点・小さな面（バッジ）のみ
- 藤色と同じく**本文テキストには使わない**。テキストに使う場合は専用の濃い階調を使う（§2 コントラスト規約）

### 装飾の制約（v0.2 で緩和）

- 装飾は「糸と結び目」「金の栞」の 2 系統に絞る。過剰な花飾りは使わない（Sayo's Journal の水彩花柄とは差別化し、こちらは「工芸の静けさ」）
- **グラデーションは面の塗りには使わない。** 例外は 2 つだけ:
  1. 写真上のスクリム（`linear-gradient(180deg, transparent 40%, rgba(43,41,38,.6) 100%)`）
  2. テキストのマーカー引き（`linear-gradient(transparent 65%, var(--color-gold-100) 65%)`）
- **和柄パターン背景は、イベントカレンダーページに限り解禁**（v0.1 の全面禁止から変更）。麻の葉・藤紫・不透明度 0.14。他ページでは使わない → §5.9

### コントラスト規約

- 本文 `#2B2926` on `#FAF7F0` = 12.9:1 ✓
- プライマリボタン: 白文字 on `#6D5BA4` = 5.6:1 ✓（AA 準拠）
- `#9A8CC4`（藤色）は**大きな装飾・アイコンのみ**に使用し、本文テキストには使わない

**★ 金の扱い（v0.2・重要）**

モックは章番号ラベル（13〜14px）に `gold-700 #A88516` を使っているが、**これは AA を満たさない**。実測値:

| 前景 | 背景 | 比 | 判定 |
|------|------|-----|------|
| `gold-600 #C9A227` | 生成り `#FAF7F0` | **2.25:1** | ✗ 大文字でも不可。テキスト使用禁止 |
| `gold-700 #A88516` | 生成り `#FAF7F0` | **3.26:1** | △ 24px 以上の大文字のみ可（AA Large 3:1） |
| `gold-800 #856911` | 生成り `#FAF7F0` | **4.82:1** | ✓ AA |
| `gold-800 #856911` | 練色 `#FBF4DC` | **4.68:1** | ✓ AA（バッジ） |

したがって:

- **小さい金の文字（章番号・英字ラベル・バッジ文字）は必ず `--color-gold-800` を使う**
- `gold-600` は**線・点・面の塗り専用**。文字色にしない
- `gold-700` は 24px 以上に限る

**朱 `#B4433A` の衝突に注意** — モックの `common.css` には案 C 用の `--color-vermilion: #B4433A` があるが、これは現行の `--color-error` と同一の値。**朱を装飾に使うとエラー表示と区別がつかなくなる**ため、v0.2 では朱を装飾色として採用しない。

**★ セマンティック色の実測（docs/17 で追加）** — 金と同じく「面の色をそのまま文字にすると AA を満たさない」ケースが 2 つあった。**面の色と文字の色を分ける**（gold-600 / gold-800 と同じ考え方）。

| 前景 | 背景 | 比 | 判定 |
|------|------|-----|------|
| `success #3D7A5C` | `success-100`（12% tint） | **4.35:1** | ✗ 12px のバッジ文字には不足 |
| `success-700 #356A50` | `success-100` | **5.07〜5.40:1** | ✓ AA（バッジ文字はこちら） |
| `ended #8A857C` | `ended-100 #EDE9DF` | **3.02:1** | ✗ 文字に使わない |
| `text-muted #6E675E` | `ended-100 #EDE9DF` | **4.60:1** | ✓ AA（終了バッジの文字はこちら） |
| `primary-400 #9A8CC4` | `bg-warm #F5EFE1` | **2.65:1** | ✗ 32px でも 3:1 に届かない |
| `primary-500 #8574B5` | `bg-warm #F5EFE1` | **3.55:1** | ✓ AA Large（§9 のプレースホルダ番号はこちら） |

## 3. タイポグラフィ

### 3.1 フォントファミリー
| ロール | 日本語 | 英語 | 備考 |
|--------|--------|------|------|
| ディスプレイ（h1/h2） | **Shippori Mincho** (600) | **Cormorant Garamond** (600) | Sayo's Journal との姉妹感。EN は locale で自動切替 |
| 見出し（h3/h4） | Shippori Mincho (500) | Cormorant Garamond (600) | |
| 本文 | **Noto Sans JP** (400) | **Inter** (400) | 読みやすさ最優先 |
| UI・メタ情報 | Noto Sans JP (500) | Inter (500) | ボタン、バッジ、日付 |

- next/font で全て self-host（サブセット化。Shippori Mincho は使用ウェイトのみ）
- `[locale]` に応じて `font-family` スタックを CSS 変数で切替（`--font-display` / `--font-body`）

**★ v0.2 で追加が必要なウェイト・スタイル**（`src/app/fonts.ts`）

v0.2 は英字イタリック（§3.3）と太い明朝を多用するが、**現行の読み込み設定では足りず、ブラウザの合成（擬似イタリック・擬似ボールド）になる**。セリフ体の合成イタリックは品質が明確に落ちるため、必ず追加する。

| フォント | 現行 | v0.2 で必要 |
|---------|------|------------|
| Cormorant Garamond | `weight: ['600']` のみ | **`style: ['normal','italic']` + `weight: ['400','500','600','700']`**（kicker・英字サブが全てイタリック） |
| Shippori Mincho | `['500','600']` | **`['500','600','700']`**（ロゴ・月見出し・スタッツ数字が 700） |
| Noto Sans JP | `['400','500']` | **`['400','500','600']`**（バッジ・チップが 600） |
| Inter | 既定 | 変更不要 |

### 3.2 タイプスケール（v0.2 で大幅拡大）

v0.1 は最大が 40px で、視線のアンカーが無く平板だった。ヒーローとセクション見出しを一段引き上げる。

| トークン | サイズ / 行間 | SP | 用途 | v0.1 |
|---------|--------------|-----|------|------|
| `display-xl` | 96px / 1.15 | 44px | **写真の無いページのヒーロー**（カレンダー等） | 新規 |
| `display` | 74px / 1.2 | 38px | **ヒーロー見出し**（ホーム・工芸詳細） | 40px |
| `h1` | 44px / 1.4 | 30px | ミッションブロック等の独立見出し | 新規 |
| `h2` | 40px / 1.3 | 28px | **セクション見出し** | 28px |
| `h3` | 22px / 1.5 | 20px | カードタイトル、工程タイトル | 20px |
| `h4` | 18px / 1.6 | — | 記事カードタイトル | 新規 |
| `lead` | 22px / 1.75 | 19px | **章の導入文**（明朝・深藤 `primary-700`） | 新規 |
| `body-lg` | 18px / 1.9 | — | 歴史・物語などの読み物本文 | 同 |
| `body` | 16px / 1.8 | — | 標準本文 | 同 |
| `caption` | 13px / 1.6 | — | メタ情報、注記 | 同 |
| `kicker` | 14px / 1.4 · `letter-spacing: .16em` | 12px | **章番号ラベル**（英字イタリック・`gold-800`） | 新規 |

- 日本語本文の行間は 1.8〜1.9 を確保（Airbnb の 1.5 前後は和文には詰まりすぎるため上書き）
- 英語ページは行間 1.6 に自動調整
- 読み物セクション（歴史・物語）の最大幅は 65ch 相当（720px）
- **見出しのサイズが上がった分、`letter-spacing: .01em〜.06em` を併用**して字間の詰まりを防ぐ（大きい明朝は字間が窮屈に見える）

### 3.3 英字併走ルール ★ v0.2 新規

v0.1 は日本語見出しが 1 行だけで階層が薄かった。v0.2 では**見出しに英字を必ず添える**。これが「編集された本」の質感を作る中核。

**3 層構造**（上から順に）

```
Chapter 01 · About                          ← kicker: 英字イタリック + 金の罫線
山藤の蔓を、糸にする。                        ← h2: 日本語・明朝
Threads spun from the wild wisteria vine.   ← 英字イタリック・鈍色
```

**英字の調達元は 3 層あり、混同しないこと。**

| 層 | 例 | 調達元 | 備考 |
|----|-----|--------|------|
| **UI 文言** | `Chapter 01` / `Book your hands-on experience.` / `Anytime` / `Contents` / 和風月名 `August · 葉月` | `messages/{ja,en}.json` | 大半はここ。追加コストなし |
| **既存 EN 訳** | 工程名 `Harvesting the wisteria vine.` / 担い手名 `The Tōyama Fuji-ito Preservation Society` | `*_translations` の `locale='en'` | **JA ページで EN 訳を読む**必要がある。かつ EN が `is_published` でないと匿名 RLS で読めない → **未取得なら英字行ごと省略**し、レイアウトが崩れない作りにする |
| **locale 非依存** | 工芸名の英字 `Tōyama Fuji-ito — wisteria-vine thread` | `crafts.name_latin`（**新規カラム**） | 詳細ページのヒーローの主役。EN 未訳でも必ず出したいので翻訳フローに乗せない |

**EN ロケール時の扱い** — EN ページでは見出しが Cormorant になるため「日本語 × 英字」の対比が成立しない。EN では **kicker（`Chapter 01 · About`）だけ残し、3 層目の英字サブタイトルは出さない**（同じ言語が 2 行続くため）。

### 4.2 面構成（背景レイヤリング）★ v0.2 新規

**v0.1 が平面的に見えた最大の原因は、全ページが生成り一色で面の切り替えが無かったこと。** 縦にスクロールするあいだに背景の面を切り替えて、リズムと奥行きを作る。

| 面 | 色 | 使いどころ |
|----|-----|-----------|
| **面 1** | 生成り `--color-bg` | 既定のページ背景 |
| **面 2** | 生成り（濃）`--color-bg-warm` | セクション帯。**全幅で敷く**（工芸グリッド、写真プレースホルダ） |
| **面 3** | 白 `--color-surface` | 独立ブロック（ミッション、カード）。上下に 1px ボーダー |
| **面 4** | 墨 `--color-text` | フッター。文字は `rgba(255,255,255,.7)`、見出しは白 |

**規約**

- 1 ページで**面 1 → 面 2 → 面 3 → 面 4 を最低 3 種類**は通過させる。生成り一色で最後まで行かない
- 面 2・面 3 は**ページ幅いっぱいに敷く**（コンテンツ幅で切らない）。内側でコンテンツ幅に絞る
- 面が変わる境界には `ThreadDivider` を置かない（面の切り替え自体が区切りになるため、二重になる）

### 4.3 余白・グリッド

- スペーシングは 4px グリッド。セクション間は 80px（SP: 56px）、面 2・面 3 のブロックは上下 96px
- **コンテンツ最大幅: 1280px**（v0.1 の 1120px から拡大。見出しが大きくなった分、余白も広げないと窮屈になる）
- カレンダーページのみ 1400px（サイドバー 380px を抱えるため）

## 7. モーション

- 基本 transition: 250ms `--ease-out`（`cubic-bezier(.2,.8,.2,1)`）
- スクロール出現: fade-in-up **0.8s / `translateY(24px)`**（v0.1 は 0.4s / 12px。ゆっくり大きく動かすほうが落ち着いて見える）。Sayo's Journal の CSS-only パターンを流用、framer-motion 不使用
- **hover の立体化**（v0.2 で強化）— v0.1 は色変化のみで平板だった

| 対象 | 動き |
|------|------|
| 工芸・記事カード | `translateY(-6px)` + `shadow-deep` + 写真 `scale(1.06)`（写真は 0.8s とゆっくり） |
| イベント行 | `translateX(4px)` + `shadow-hover` + 左罫が金に |
| Primary ボタン | `translateY(-1px)` + 藤色の影 |
| 円形矢印ボタン | 背景が淡藤 → 藤紫に反転 + `translateX(4px)` |
| ヒーロー写真 | hover で 8s かけて `scale(1.05)`（気づかない速度で「生きている」感じを出す） |

- `prefers-reduced-motion: reduce` で全アニメーション・トランジションを無効化（**新規の transform 系も必ず含める**）

## 8. レスポンシブ / アクセシビリティ

- ブレークポイント: SP < 640 / Tablet 640–1024 / PC > 1024
- タップターゲット 44px 以上（高齢ユーザー・観光中の片手操作を考慮）
- フォーカスリング: 藤紫 2px（`:focus-visible`）
- 画像は必ず alt（翻訳テーブルの locale 対応 alt）
- 日付は locale フォーマット（ja: 8月23日(土) / en: Sat, Aug 23）

### ★ v0.2 のモバイル設計（モックに無い部分）

**モックはすべて `min-width: 1200px` のデスクトップ専用で、SP レイアウトは存在しなかった。** 以下を実装側の規約とする。

| 要素 | PC | SP |
|------|-----|-----|
| ヒーロー（ホーム） | 2 カラム（コピー / 写真） | 1 カラム。**写真を先に**置き、その下にコピー |
| `display` 見出し | 74px / 96px | 38px / 44px（§3 のスケール表） |
| 工芸詳細の 3 カラム | 目次 / 本文 / サイドバー | 1 カラム。目次は**上部の横スクロールチップ**、サイドバーカードは「体験する」の位置にインライン展開（追従しない） |
| カレンダーの 2 カラム | リスト / サイドバー 380px | 1 カラム。**サイドバーはリストの下**へ（随時受付 → ミニカレンダーの順） |
| カードグリッド | 3 列 | Tablet 2 列 / SP 1 列 |
| イベント行 | 日付 / 本文 / 写真 / 料金の 4 カラム | 日付 + 本文の 2 カラムに縮約し、**写真と料金は本文の下**へ折り返す |
| スタッツ帯 | 4 列 | 2 列 × 2 段 |
| 月カレンダー | セル最小高 120px + イベント名 | セルは日付と結び目ドットのみ。**タップで下にその日のリストを開く** |
| フッター 4 カラム | 4 列 | 1 列に積む（ブランド → コンテンツ → について → 言語） |

- **横スクロールを発生させない。** `min-width` を指定しない、`grid-template-columns` は必ず `repeat(auto-fit, ...)` かメディアクエリで畳む
- 麻の葉パターン（§5.9）は SP でも維持してよいが、`background-attachment: fixed` は iOS で描画が重いため **SP では `scroll` に落とす**
- 追従要素（目次・サイドバー）は SP では `position: static` に戻す。画面高が足りず操作を阻害するため

## 9. 写真ガイドライン（紗代さん向け運用メモ）

- イラストではなく実写真のみ。手元・素材・工程のクローズアップを優先
- カード用は 4:3、Hero 用は 16:9 で撮影 or トリミング
- 人物が写る場合は掲載許可を撮影時に確認（交渉時のチェック項目に含める）
- 明るめ・自然光。生成り背景と喧嘩する強い彩度の加工は避ける

### 必要なアスペクト比（v0.2）

| 用途 | 比 | 備考 |
|------|-----|------|
| ホーム Hero | 4:5（縦） | v0.2 で縦長に変更。人物・手元の縦構図が映える |
| 工芸詳細 Hero | 16:8 | 横長。タイトルを被せるので**下 1/3 に主題を置かない** |
| 工芸カード | 4:3 | |
| 工程写真 | 4:3 | 工程数分（1 工芸あたり 4〜6 枚） |
| 記事カード | 16:10 | |
| イベント行サムネ | 3:2 | 小さいので寄りの構図で |

### ★ 写真が無いあいだの見せ方（重要）

**掲載交渉が終わるまで他者の写真は一切使わない**（REQUIREMENTS.md §10）。現在シードは**全件写真ゼロ**で、docs/15 まで入らない。v0.2 は写真主導のデザインなので、**プレースホルダの設計が実質的なデザインの一部**になる。

- 基本形: `bg-warm` の面 + `border-strong` の**破線 1px** + 中央に英字の通し番号（**`primary-500`**・32px）+ 工芸名 / 状態（「掲載交渉中」等）
  - 通し番号は当初 `primary-400` としていたが `bg-warm` 上で 2.65:1 と大きな文字の 3:1 に届かないため、docs/17 で `primary-500`（3.55:1）に変更した
- **グレーの箱にしない。** 生成り系の面と破線で「これから入る枠」であることを示す
- ヒーローなど大面積のプレースホルダには**麻の葉パターンを敷いてよい**（§5.9 の例外。空白の面積が大きすぎるため）
- 写真が無い状態でもレイアウトが破綻しないこと。**写真の有無で行の高さが変わらない**よう、アスペクト比は常に固定する
- **モック内の画像は外部ホストの仮素材で、出典・ライセンスとも不明。実装に持ち込まない**
