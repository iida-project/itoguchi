## 1. 背景と目的

### 課題
南信州（飯田・下伊那）の伝統工芸の情報は、個別メーカーのサイト、無料ブログサービス、観光サイトの断片的な紹介、SNS 投稿に散在しており、体系的にたどり着ける「正本」が存在しない。

- 全国区の工芸ポータルは国指定の伝統的工芸品が中心で、飯田水引・遠山ふじ糸・阿島傘などの地域工芸は掲載対象外か情報が薄い
- 各会・工房の発信は Jimdo 無料ブログや SNS が中心で、情報が時系列に流れて埋もれる
- 「体験してみたい」と思っても、いつ・どこで・いくらで・どう申し込むのかが 1 か所でわからない
- SNS はログイン壁の中にあり、検索エンジンにも AI にも到達不能

### サイトの役割 = 体験への案内係
本サイトは工芸の百科事典ではなく、**「体験したい人が、このサイトを見れば『どうすればできるか』がわかる案内係（コンシェルジュ）」**である。

- 入口は「知る」だけでなく「体験する」「行く」
- 各工芸のワークショップ・イベントを**カレンダーで横断的に一覧**できる
- 常時申込可能な体験（随時受付）と、日付が決まったイベントを区別して案内する
- 申込・問い合わせは各会へ誘導（予約システムは持たない）

### 目的
1. **点を線に**: 散在する南信州の伝統工芸情報を、工芸単位の「正本ページ」と横断カレンダーに集約する
2. **体験への最短経路**: 「体験したい」→「申し込み方がわかる」までをこのサイトだけで完結させる
3. **AI に見つかる地域情報**: 構造化データを備えた公開 Web として、検索エンジンと AI（AIO/LLMO）の両方から参照される状態を作る
4. **リニア時代への備え**: 県外来訪者・訪日客が増える将来に向け、日英 2 言語で「南信州の伝統工芸の入口」を先に用意する
5. **モデルの横展開証明**: Sayo's Journal で構築した「取材 → 記事化 → 構造化 → 循環」モデルが、テーマ特化型サイトとして複製可能であることをビジコン文脈で実証する

### Sayo's Journal との関係
- Sayo's Journal = 地域の「今」を扱うフロー型メディア（記事が主役）
- 本サイト = 工芸の「体系」と「体験導線」を扱うストック型ポータル（**工芸エンティティ + 体験・イベントが主役**）
- 技術構成・管理パネルの設計資産は流用する（§9）
- 相互リンク: Sayo's Journal の関連記事 ↔ 本サイトの工芸ページ

## 4. スコープ

### 作るもの（MVP / Phase 1）
- 公開サイト（日英 2 言語）
  - ホーム（「体験する」導線を最上位に）
  - 工芸一覧
  - **工芸詳細（正本ページ）**: 概要 / 歴史・物語 / 工程 / 担い手紹介 / 体験情報 / 見られる・買える場所 / 関連記事 / アクセス
  - **体験一覧**: 随時受付の体験プログラムを横断一覧（工芸・地域・受付形態の絞り込みチップ付き）
  - **イベントカレンダー**: 日付が決まったワークショップ・イベントを月表示 + リスト表示で横断一覧（工芸での絞り込み付き）★案内係の中核
  - 記事一覧・記事詳細（紗代さんの取材記事）
  - About（サイトの目的・運営者）
- 管理パネル（工芸・記事・画像・体験・イベントの CRUD、日英両方の編集）
- JSON-LD 構造化データ / サイトマップ / OGP
- AI 下訳パイプライン（日本語正本 → Gemini 下訳 → 紗代さんレビュー → EN 公開）

### カレンダーの設計方針（掲載数が少ない時期への対策）
- 初期は 2 工芸のため、月間イベント数が少なくカレンダーが空白だらけになるリスクがある
- 対策: **リスト表示（開催予定順）を主、月カレンダーを従**とし、切替可能にする
- イベントが無い月でも「随時受付の体験」を常に併記し、空白ページを作らない
- イベント登録が増えてきたらカレンダー表示を前面に出す（表示切替はフラグで制御）

### 作らないもの（MVP では見送り）
- 情報窓口フォーム（各会からの更新受付）→ Phase 2
- EC / 販売機能（購入場所への外部リンクのみ）
- 予約システム（体験・イベントの申込は各会の連絡先・申込先へ誘導）
- AI チャット → Phase 3 検討
- 日英以外の言語（構造上は追加可能にしておく）

## 5. 多言語方針（i18n）

- **対応言語**: 日本語（正本）+ 英語（MVP から公開）。locale 追加可能な構造にする
- **ルーティング**: App Router の `[locale]` セグメント方式（`/ja/...` `/en/...`、`/` は Accept-Language でリダイレクト）
- **翻訳ワークフロー**:
  1. 紗代さんが日本語で執筆・入稿
  2. 管理パネルの「英訳生成」ボタンで Gemini が下訳を生成（工芸用語の対訳集 glossary をプロンプトに注入）
  3. 紗代さんがレビュー・修正して EN 公開フラグを ON
  4. 日本語のみ公開 / 日英公開 の状態を工芸・記事・イベント単位で持てる（EN 未訳でもサイトは壊れない）
- **翻訳データの持ち方**: 翻訳テーブル方式（`*_translations` に locale ごとの行）
- **固有名詞**: ローマ字 + 補足方式（例: Tōyama Fuji-ito (wisteria-vine thread)）。glossary テーブルで一元管理
- **UI 文言**: next-intl のメッセージファイル（ja.json / en.json）

## 6. 画面構成（公開側）

```
/[locale]                       → ホーム（Hero + 体験・イベント直近 + 工芸カード + 最新記事）
/[locale]/crafts                → 工芸一覧
/[locale]/crafts/[slug]         → 工芸詳細（正本ページ）
/[locale]/experiences           → 体験一覧（随時受付の体験プログラム）
/[locale]/events                → イベントカレンダー（月表示 ⇄ リスト表示）★案内係の中核
/[locale]/events/[slug]         → イベント詳細（日時・場所・料金・申込方法）
/[locale]/articles              → 記事一覧（工芸・タグでフィルタ）
/[locale]/articles/[slug]       → 記事詳細
/[locale]/about                 → About
/[locale]/privacy               → プライバシーポリシー
```

### 工芸詳細ページの構成
1. Hero（代表写真 + 工芸名 + ひとこと）
2. 概要（何であるか、3〜4 文）
3. 歴史・物語（藤姫物語のようなナラティブを活かすセクション）
4. 工程（ステップ形式、写真付き）
5. 担い手（会・工房の紹介、人の顔が見える）
6. 体験する（随時体験 + 直近イベント。「どうすれば体験できるか」を明記）★
7. 見る・買う（場所リスト + 地図）
8. 関連記事（取材記事カード）
9. アクセス・問い合わせ

### イベント詳細ページの構成（案内係として必須の項目）
日時 / 会場（地図）/ 料金 / 定員・対象 / 持ち物 / 申込方法（外部リンク or 電話・メール）/ 主催（担い手ページへ）/ 関連工芸

## 7. データモデル（概要）

```
crafts                 -- 工芸（正本エンティティ）
  id, slug, status(draft/published), hero_image_url,
  region, created_at, updated_at
craft_translations     -- locale, name, tagline, overview, history,
                          is_published（locale 単位の公開制御）
craft_steps            -- 工程（order, image_url）
craft_step_translations
groups                 -- 担い手（会・工房）
  slug, address, lat/lng, contact, sns_urls[]
group_translations
experiences            -- 随時受付の体験プログラム
  craft_id, group_id, availability(anytime/seasonal/request),
  price_note, duration_note, season_note, apply_method
experience_translations
events                 -- 日付が決まったイベント・ワークショップ ★MVP に含める
  craft_id, group_id, slug, status(draft/published/ended),
  start_date, end_date, time_note(自由記述),
  venue, address, lat/lng, fee_note, capacity_note,
  apply_url, apply_note, created_at, updated_at
event_translations     -- locale, title, description, is_published
spots                  -- 見られる・買える場所
  craft_id, name, type(shop/museum/other), address, lat/lng, url
spot_translations
articles               -- 取材記事
  craft_id(FK), slug, thumbnail, published_at
article_translations   -- locale, title, content(HTML), excerpt
glossary               -- 工芸用語の対訳集（ja, en, note）
```

方針:
- 翻訳は「本体テーブル + translations テーブル」で統一（locale, is_published を translations 側に持つ）
- events の日付・会場系フィールドは Sayo's Journal posts の event_* フィールド群（is_event, event_date_start/end, event_time_start/end, event_venue/address/fee/url）の設計を踏襲。時刻は自由記述（「10時頃〜」等に対応）
- 終了イベントは自動で ended 扱い（end_date < 今日）とし、アーカイブとして記録に残す（削除しない）— 「活動の記録が残る」価値は Sayo's Journal と共通
- RLS: 公開テーブルは匿名 SELECT（published のみ）、書き込みは認証のみ

## 8. SEO / AIO

- JSON-LD: `Article` / `BreadcrumbList` / `WebSite` / `Organization` に加え、工芸ページに `TouristAttraction` + `Place`、**イベントに `Event`（開催日・場所・料金入り）**、担い手に `Organization` / `LocalBusiness` を検討（実装時に精査）
- イベントの `Event` 構造化データは Google のイベントリッチリザルト対象 → 「飯田 体験 ワークショップ」系の検索面で露出が狙える
- hreflang（ja / en / x-default）を全ページに出力
- サイトマップ動的生成（locale × ページ）
- 工芸詳細・体験は ISR、カレンダーは日次更新で十分（force-dynamic 不要）
- 「南信州 伝統工芸 体験」「Minami-Shinshu craft workshop」「遠山 藤糸」「阿島傘」等のクエリで正本として参照されることを成功指標とする

## 10. 交渉用デモ（プレビュー版）の方針

MVP 実装時、掲載交渉前の 2 団体（遠山ふじ糸伝承の会 / 阿島傘）のコンテンツは**仮内容（シードデータ）として開発側で用意**し、完成イメージを交渉材料として提示する。

### シードコンテンツのルール
1. **文章**: 公開情報（公式サイト・公開されている紹介）を基にドラフトを作成。推測で補った箇所（料金・頻度・定員等）は `is_provisional` フラグ + 本文中の仮表示（薄字 + 「※確認中」）でマークし、**交渉時の確認リストとして流用**する
2. **写真**: 許可取得前は他者の写真を一切使用しない。プレースホルダー枠（生成り背景 + 「ここに◯◯の写真が入ります」キャプション）で構成し、「あなたの写真でここが完成します」という提案として機能させる
3. **公開制御**: 交渉期間中は本番公開しない
   - 全ページ `noindex, nofollow` + robots.txt で拒否
   - Vercel プレビュー URL（またはパスワード保護）で共有
   - 許可・素材の揃った団体から順に craft 単位で本公開へ切替（crafts.status で制御）
4. **交渉成立後**: 提供された写真・確認済みテキストで置換 → is_provisional 解除 → 公開。掲載許可の記録（誰から・いつ・範囲）を管理パネルのメモ欄に残す

### 交渉時の確認チェックリスト（シードから自動生成）
- 掲載可否（団体紹介 / 歴史・物語 / 工程 / 体験情報）
- 写真提供の範囲と提供方法（人物写り込みの扱い含む）
- 体験・イベント情報の粒度（料金・定員・申込方法をどこまで載せるか）
- 問い合わせ先として掲載してよい連絡手段
- 英語ページ掲載の可否
