## 運用ルール

- 各チケット内の Todo は `- [ ]` で管理し、完了したら `- [x]` に更新する
- チケット全体が完了したら、この表の「状態」を「完了」に更新する
- 仕様に迷ったら REQUIREMENTS.md（要件）と DESIGN.md（デザイン）が正。チケットはそれらの分割ビュー
- 実装中に判明した仕様変更・未決事項は該当チケットの「メモ」節に追記する
