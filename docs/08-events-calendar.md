# 08 — イベントカレンダー・イベント詳細 ★案内係の中核

**依存**: 02, 04
**参照**: REQUIREMENTS.md §4（カレンダー設計方針）§6 / DESIGN.md §5.3, §6

## 概要

`/[locale]/events`（月表示 ⇄ リスト表示）と `/[locale]/events/[slug]`。本サイトの中核機能。

## 仕様

### 一覧
- **リスト表示が主・月カレンダーが従**。ヘッダーで切替（前面に出す方をフラグで制御できる構造に）
- リスト行: 左に日付ブロック（淡藤背景 + 藤紫の日付数字）、右に タイトル + 受付中バッジ / 会場 / 料金 / 工芸名リンク
- 月カレンダー: 自作グリッド（ライブラリ不使用）。イベント日に藤紫の結び目ドット、タップで当日イベントをリスト表示。月移動は左右矢印 + 「今日」ボタン
- 工芸での絞り込み
- **空の状態**: 「今月の開催予定はありません」で終わらせず、随時受付の体験カードを必ず併記
- 終了イベント: 写真グレースケール + 「終了」バッジでアーカイブ表示（削除しない）
- カレンダーは日次更新で十分（ISR、force-dynamic 不要）

### 詳細（案内係として必須の項目）
日時 / 会場（地図）/ 料金 / 定員・対象 / 持ち物 / **申込方法**（外部リンク or 電話・メール。ボタンラベルは「申し込む（外部サイト）」等、外部遷移を明示）/ 主催（担い手へ）/ 関連工芸

## メモ

- 初期表示（リスト主 or 月表示主）の最終判断はイベント登録数を見て決定（REQUIREMENTS.md §13）。まずリスト主で実装
- **ISR × 切替/絞り込み**: docs/07 と同方式。全件取得はサーバー、リスト⇄カレンダー切替・工芸絞り込み・カレンダー操作はクライアント（`EventsView`/`MonthCalendar`、`'use client'`）。`useSearchParams` で初期化→ `history.replaceState` で `?view=&craft=` 同期（共有可能・RSC 再取得なし）。`useSearchParams` のため `<Suspense fallback={開催予定リスト}>` でラップ＝プリレンダリング HTML に開催予定が載る（SEO）。ページは ● SSG のまま
- **月カレンダー自作**（ライブラリ不使用・`MonthCalendar.tsx`）: 曜日/月見出しは `useLocale()`＋`Intl` 生成（メッセージキー不要）。日付判定は 'YYYY-MM-DD' 辞書順比較。イベント日に結び目ドット（開催予定=藤紫 / 終了のみ=グレー）。日セル 44px 以上。`today` は server の `todayISO()` を渡してハイドレーション不一致を回避
- **空月→随時体験併記**: リスト（開催予定 0 件）・カレンダー（その月にイベント無し）の両方で `ExperienceCard` を併記（空白ページ禁止）。今日が 2026-07 で 7 月は空月のため既定カレンダーで体験カードが出る
- **EventRow 改修**: 任意 prop `craft?: {slug,name}` を追加。行全体リンク（タイトル Link の `::after`）＋工芸リンク（`z-10`）でアンカー入れ子を回避。ホーム/工芸詳細の既存利用は後方互換
- **詳細ページ申込**: `apply_url` があれば外部ボタン「申し込む（外部サイト）↗」（`target=_blank rel=noopener`、`buttonClasses('primary')`）、`apply_note` はテキスト併記。**持ち物**は専用フィールドがスキーマに無いため省略（description で表現する運用）。`Event` JSON-LD は docs/14 に委譲（必要データは詳細ページで揃えた）
- **seed（demo 拡充）**: アーカイブ確認用に過去イベント `fuji-ito-workshop-2026-06`（published・2026-06-15）を追加（→ 派生 ended）。外部申込ボタン確認用に ajima イベント（9/20）へプレースホルダ `apply_url=https://example.com/apply`（※確認中）を付与。既存の仮電話/料金と同格の暫定値で、§10 の写真規定には抵触しない。events 計 3 件（過去1/未来2）
