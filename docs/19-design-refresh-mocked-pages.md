# 19 — デザインリフレッシュ: モックのある 3 画面

**依存**: 17（工芸詳細は 18 も）
**参照**: DESIGN.md v0.2 §6 / モック `docs/design/modern/{home,craft-detail,events}.html`

## 概要

モックが存在する 3 画面（ホーム / 工芸詳細 / イベントカレンダー）を v0.2 に作り替える。**この 3 画面が残り 7 画面の型になる**ので、ここで作った部品は再利用前提で切り出す。

**モックはデスクトップ専用（`min-width: 1200px`）。SP レイアウトは DESIGN §8 の規約に従って実装側で設計する。**

## 仕様

### 共通

- 全セクション見出しを `SectionHeading`（§5.0）に統一
- 面構成（§4.2）を必ず適用。**1 ページで最低 3 種類の面を通過**させる
- モック内の画像は外部ホストの仮素材。**持ち込まない**。写真は全て §9 のプレースホルダで表現する
- ISR の設定（`revalidate` / `dynamicParams` / `generateStaticParams`）は現行のまま維持
- 既存の「ISR × インタラクティブ絞り込み」パターン（サーバーで全件取得 → `'use client'` が `useSearchParams` で初期化 → `history.replaceState` で URL 同期 → `<Suspense fallback={全件}>`）を**壊さない**

### ホーム（§6 ホーム）

面の順序: **1 → 1 → 1 → 2 → 3 → 1 → 4**

1. **Hero を 2 カラム化**（コピー 1.05fr / 写真 .95fr・4:5・`shadow-deep`）。kicker → `display` 74px（強調語は藤紫 + 金の下線）→ 英字 26px → リード → CTA 2 つ（`lg`）
2. **スタッツ帯**（`Stat`）を追加。上に 1px ボーダー
3. 直近の体験・イベント（`EventRow` に円形矢印を追加）
4. **工芸グリッドを面 2（`bg-warm`）で全幅に敷く**。カードに `No. 01` の通し番号
5. **ミッションを面 3（白）の全幅ブロックに**。図を縦 3 行リスト → **横一直線の糸**（工房サイト → SNS → **いとぐち**（金・大）→ 検索・AI → 体験する人）に変更
6. 最新記事 3 列。タグを英字イタリック（`Interview · 遠山ふじ糸`）に

### 工芸詳細（§6 工芸詳細）

- **3 カラム化**（目次 240px / 本文 1fr / 追従サイドバー 300px・gap 60px）
- **Hero**: 16:8 大写真 + スクリム + タイトル被せ。左上に `No. 01 · {name_latin}` プレート、左下に工芸名（74px 白）+ 英字 + タグライン、右下にメタ 4 項目（Origin / Region / Makers / Experiences）
- **`CraftToc`**: 番号を `decimal-leading-zero`（`01 02 03…`）に。現在地ハイライトの既存ロジックは維持
- **追従サイドバーカード**（§5.12）を新規作成。直近 2〜3 件 + 「すべての体験を見る」
  - **本文中の「体験する」セクションは残す**（役割分担: サイドバー = いつでも申し込める入口 / 本文 = 詳細一覧）
- **章見出しを 3 層に**（docs/18 のデータを使用）。01・02 は DB、03 は工程数、04 は担い手名、05 は固定
- 歴史・物語を**引用ボックス**（淡藤面 + 明朝 17px `primary-700` + 開き括弧装飾）に
- `StepTimeline` の結び目を**藤紫の芯 + 生成りの縁 + 金の外輪**に。各工程を `Step 01 · 採取` → 工程名 → 英字サブ → 説明の 4 層に

### イベントカレンダー（§6 イベントカレンダー）

- **`<body>` に麻の葉パターン**（§5.9）。**このページだけ**。SP では `background-attachment: scroll` に落とす
- ページ Hero: `display-xl` 96px（強調語に金のマーカー引き）+ 英字 24px。右にスタッツ帯をカード化
- フィルタチップに**件数バッジ**。工芸で絞る群と状態で絞る群のあいだに間隔
- **2 カラム化**（リスト 1fr / サイドバー 380px）。サイドバーは常設で「随時受付の体験」+ ミニカレンダー
- リストを**月グルーピング**（`8月 / August · 葉月` + `02 件の開催`）。終了イベントは末尾に「終了 / Past events · アーカイブ」でまとめる
- `EventRow` に**左罫 4px**（藤紫 → hover で金 + `translateX(4px)`）。右端に料金と申込リンクを独立
- `MonthCalendar`: 結び目ドットを色分け（金 = 受付中 / 藤紫 = 要予約 / 鈍色 = 終了）。セル最小高 120px でイベント名を直接表示
- カレンダー表示に切り替えたら**全幅**（サイドバーを畳む）。下に「開催予定がない日でも随時受付の体験があります」の淡藤ボックス
