## 概要

モックが存在した 3 画面（ホーム / 工芸詳細 / イベントカレンダー）を v0.2 に作り替える。**この 3 画面が残り 7 画面の型になる**ので、ここで作った部品は再利用前提で切り出す。

**モックはデスクトップ専用（`min-width: 1200px`）だった。SP レイアウトは DESIGN §8 の規約に従って実装側で設計する。**

## 仕様

### 共通

- 全セクション見出しを `SectionHeading`（§5.0）に統一
- 面構成（§4.2）を必ず適用。**1 ページで最低 3 種類の面を通過**させる
- 写真は 1 枚も無い。全て DESIGN §9 のプレースホルダで表現する（`bg-warm` + 破線 + 英字通し番号。グレーの箱にしない）
- ISR の設定（`revalidate` / `dynamicParams` / `generateStaticParams`）は現行のまま維持
- 既存の「ISR × インタラクティブ絞り込み」パターン（サーバーで全件取得 → `'use client'` が `useSearchParams` で初期化 → `history.replaceState` で URL 同期 → `<Suspense fallback={全件}>`）を**壊さない**

### ホーム（§6 ホーム）

面の順序: **1 → 1 → 1 → 2 → 3 → 1 → 4**

1. **Hero を 2 カラム化**（コピー 1.05fr / 写真 .95fr・4:5・`shadow-deep`）。kicker → `display` 74px（強調語は藤紫 + 金の下線）→ 英字 26px → リード → CTA 2 つ（`lg`）

3. 直近の体験・イベント（`EventRow` に円形矢印を追加）
4. **工芸グリッドを面 2（`bg-warm`）で全幅に敷く**。カードに `No. 01` の通し番号
5. **ミッションを面 3（白）の全幅ブロックに**。図を縦 3 行リスト → **横一直線の糸**（工房サイト → SNS → **いとぐち**（金・大）→ 検索・AI → 体験する人）に変更
6. 最新記事 3 列。タグを英字イタリック（`Interview · 遠山ふじ糸`）に

### 工芸詳細（§6 工芸詳細）

  - **本文中の「体験する」セクションは残す**（役割分担: サイドバー = いつでも申し込める入口 / 本文 = 詳細一覧）

### イベントカレンダー（§6 イベントカレンダー）

- **`<body>` に麻の葉パターン**（§5.9）。**このページだけ**。SP では `background-attachment: scroll` に落とす

- **2 カラム化**（リスト 1fr / サイドバー 380px）。サイドバーは常設で「随時受付の体験」+ ミニカレンダー
- リストを**月グルーピング**（`8月 / August · 葉月` + `02 件の開催`）。終了イベントは末尾に「終了 / Past events · アーカイブ」でまとめる

### データの制約でモックどおりに作れなかった 5 点（**新カラムは足していない**）

18 で「追加カラムは 3 本に抑える」と決めた直後なので、見た目のためにスキーマを増やさない。出せないものは英字行と同じく「無ければ詰める」で処理した。

| モックの要素 | 実データ | 対応 |
|---|---|---|
| Hero メタの `Origin 17c〜` | 創業年のカラムが無い | **Origin を出さない**（Region / Makers / Experiences の 3 項目） |
| `受付中` `残席2` バッジ | 受付状態・残席のカラムが無い | 出さない。**状態チップも「開催予定 / 終了」の 2 つだけ** |
| 記事タグ `Interview · 遠山ふじ糸` | 記事にカテゴリ列が無い | 英字イタリックの**工芸名のみ**（`nameLatin · name`） |
| カレンダーのドット 3 色 | 「要予約」の区別が無い | 金（開催予定）/ 鈍色（終了）の 2 色 |
| 全ページの写真 | 全件ゼロ | §9 のプレースホルダ。**アスペクト比を固定**して写真の有無で高さを変えない |

### 実装で決めたこと

- **工芸詳細 Hero の写真ゼロ時**: 枠（16:8）は保ったまま `bg-warm` + 破線 + **麻の葉**（§9 が大面積プレースホルダに認めた例外）にし、文字は墨。写真が入ったらスクリム + 白文字に変わるだけでレイアウトは動かない
- **SP の追従サイドバー**: `hidden lg:block`（PC）と `lg:hidden`（SP・「体験する」セクション内にインライン）の**出し分け**にした。DOM は重複するが常にどちらか一方だけが表示される。grid の子は本文全体で 1 つなので、order では本文の途中に差し込めないため
- **ホーム Hero の改行位置**は文言側で決める（`<br></br>` + `t.rich`）。和文は任意の位置で折り返るため、「手しごと|に、」のような割れ方を防ぐ
- **カレンダー Hero の見出し**: JA は「開催の**暦**。」+ 英字サブ、EN は英字サブの文をそのまま見出しにして 3 層目を出さない（§3.3 の EN 規約と整合）
- **和風月名は EN でもローマ字併記**（`August · Hazuki`）。DESIGN §10 の未決事項をここで決定し、§10 に追記済み
- ホーム Hero のリード文は**現行のコピーのまま**。レイアウトだけ v0.2 にした（コピー確定は §10 の未決事項に残る）

### 直したバグ（v0.1 から潜在していたもの）

`formatDateParts` の `day` が **ja で `23日` を返していた**（`Intl` の `{day:'numeric'}` は ja-JP で単位が付く）。日付ブロックが「8月 / 23 / 日 / 日」と 4 行になっていた。数字表記が同じ `en-US` で整形するよう修正。
